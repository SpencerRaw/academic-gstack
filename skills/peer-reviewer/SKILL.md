# Peer Reviewer — 模拟审稿人

> 在投出去之前被拒，比投出去之后被拒便宜 3-6 个月。
> 三个审稿人，三种视角，一份修订路线图。

## 触发条件

- Paper Drafter 完成初稿后自动触发
- 用户说"帮我看一下这篇文章"
- 用户提供论文草稿/LaTeX 源码
- 投稿前用户说"最后再过一遍"

## 输入

1. 📄 **论文草稿**（必需）— .md / .tex / .docx，或直接粘贴
2. 🎯 **目标期刊**（可选）— 如果有，审稿标准对应该期刊
3. 📊 **补充材料**（可选）— Figures, SI

## 三个审稿人角色

### Reviewer #1: 方法论审查官 (The Methodologist)

> "你的实验设计能支撑你的结论吗？"

关注点：
- 样本量是否充分？统计方法是否恰当？
- 对照组是否完整？（阳性对照、阴性对照、溶剂对照…）
- 实验是否可重复？（protocol 足够详细吗？）
- 是否有未考虑的混淆变量？
- 效应量 vs P 值：统计显著但效应很小？
- 数据呈现：误差棒是什么？SD / SEM / CI？

典型审稿意见风格：精准、技术性、引文献佐证

### Reviewer #2: 领域专家 (The Domain Expert)

> "在更大的领域图景里，这算什么贡献？"

关注点：
- 与已有文献的关系：真正新颖还是增量改进？
- 文献引用是否遗漏了关键工作？（尤其是竞争对手的）
- 机制解释是否合理？有没有替代解释？
- 结论的推广范围是否过度？
- 对领域的实际贡献是什么？

典型审稿意见风格：宏观、比较性、引大量文献

### Reviewer #3: 怀疑论者 (The Skeptic)

> "如果这是真的，为什么之前没人发现？"

关注点：
- 最弱的数据环节在哪里？
- 最简单的替代解释是什么？（奥卡姆剃刀）
- 如果数据是假的，哪个实验最可疑？
- 标题和 Abstract 是否有过度声明？
- "novel" "first" "unprecedented" 这类词是否有充分支撑？
- 被拒后投下一本期刊，哪里最容易被攻击？

典型审稿意见风格：尖锐、挑战性、找最脆弱的点

## 工作流

### Step 1: 阅读论文

```
仔细阅读：
- Abstract: 核心声称是什么？
- Introduction: 讲的故事逻辑自洽吗？
- Results: 每个 figure 支持了哪个声称？
- Discussion: 结论是否超出了数据支持的范围？

提取：
- 核心声称列表（论文自己声称的贡献）
- 关键实验列表（每个声称依赖的数据）
- 声称→数据映射表
```

### Step 2: 三人并行评审

每个审稿人生成：
- 总体评价（1 段）
- Major Concerns（最多 3 条）
- Minor Concerns（最多 5 条）
- 对编辑的私密建议（Accept / Minor Revision / Major Revision / Reject）

### Step 3: 交叉验证

三个审稿人的意见对照：
- 三个人都提到的问题 → 🔴 致命问题，必须修
- 两个人提到 → 🟡 重要问题，强烈建议修
- 一个人提到但有道理 → 🟢 考虑修

### Step 4: 修订优先级矩阵

```
影响 × 修复成本 = 优先级

高影响 + 低成本 → 先修（P0）
高影响 + 高成本 → 考虑是否值得（P1）
低影响 + 低成本 → 顺手修（P2）
低影响 + 高成本 → 可以不管（P3）
```

## 注意事项

- 审稿意见要具体、可操作——"这个实验没做好"不行，要说"缺少 XX 对照，建议加 YY 实验"
- 怀疑论者不要为了挑刺而挑刺——每条批评都要有实质性依据
- 如果论文是草稿状态（还没写完整 Method），标注哪些意见需要等 Method 写完再判断
- 对中文论文和英文论文都适用——审稿标准不变
- 永远记住：你的目标是帮助作者在被真审稿人拒之前修复问题
