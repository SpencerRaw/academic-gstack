# Lit Reviewer — 文献审查员

> 你不是在"搜论文"——你是在为下游 Hypothesis Generator 准备原材料。
> 输出质量直接决定假说生成的质量。

## 触发条件

- 用户提供研究课题 + 关键词
- 用户说"查一下 X 领域最新进展"
- 夜间流水线自动触发（cron job）
- 用户说 `/lit-review` 或提及本 skill

## 输入格式

用户提供（或从上下文推断）：
```
课题: [一句话描述]
关键词: [逗号分隔，中英文都可]
数据库: [arXiv/bioRxiv/PubMed/Google Scholar，默认 arXiv+bioRxiv]
时间范围: [默认近2年]
筛选标准: [默认：研究性论文，排除综述/预印本v1]
```

## 工作流

### Phase 1: 检索 (web_search)

```
对每个数据库，用关键词组合检索：

1. 主检索："{关键词1} {关键词2}" → 获取 5 篇最相关
2. 扩展检索："{关键词1} AND ({关键词3} OR {关键词4})" → 5 篇
3. 前沿检索："{关键词1}" + site:arxiv.org + 近3个月 → 5 篇
4. 高引检索："{关键词1}" + 被引>50 → 5 篇（经典文献）
```

每步调用 `web_search(limit=5)`，去重合并，目标得到 15-25 篇候选论文。

### Phase 2: 提取 (web_extract)

对每篇候选论文，用 `web_extract(url)` 提取摘要/全文。

关键信息提取：
- 研究问题（他们想解决什么？）
- 核心假说（他们提出了什么机制？）
- 关键实验（他们做了什么来验证？）
- 主要结论（数据支持了什么？）
- 局限性（作者自述 + 你判断的）

### Phase 3: 分类排序

按以下维度评分（1-10）：

| 维度 | 权重 | 标准 |
|------|------|------|
| 相关性 | ×3 | 与课题直接相关程度 |
| 新颖性 | ×2 | 方法/发现的创新程度 |
| 可复现性 | ×2 | 方法描述是否清晰可复现 |
| 影响力 | ×1 | 期刊 IF / 引用数 / 领域关注度 |
| 假说密度 | ×1 | 能提取多少可检验假说 |

综合分 = Σ(维度分 × 权重)，降序排列。

### Phase 4: 假说提取

对 Top 5 论文，逐篇提取：

1. **作者的核心假说**: 他们明确提出的因果/机制假说
2. **未检验的推论**: 从他们数据中能逻辑推导但未验证的假说
3. **与你的课题的交叉点**: 这些假说如何连接到你的研究方向

### Phase 5: 空白标注

分析 Top 10 论文的覆盖范围，标注：
- ✅ 已有充分证据的
- ⚠️ 有初步证据但矛盾的
- ❌ 完全空白的（这是你的机会）

## 注意事项

- **不要编造论文**。每篇论文必须有真实 URL，通过 web_search/web_extract 获取
- 假说提取要从论文数据出发，不是从你的期望出发
- 如果某领域论文太少（<10 篇），扩大检索范围或更换关键词
- 中文论文也可以用，但优先英文（尤其是顶刊）
- 每篇论文标注"对哪个在研课题有用"
